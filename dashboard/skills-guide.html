The WHAT

Claude Skills are reusable instruction files that live alongside your conversations. Instead of re-explaining your preferences, coding standards, or workflows every time, you write a Skill once — and Claude reads it automatically whenever it's relevant. Think of them as persistent expertise that Claude can tap into on demand.

Unclog Your Context

Every token spent on instructions is a token stolen from your actual conversation.

The WHY

System prompts are loaded into your conversation every single time you send a message. As your conversation grows, they compete with your actual messages for space in the context window. Skills are different — Claude reads the Skill file only when relevant, and the instructions don't bloat your context.

Context Window
System Prompt ~120,000 tokens
↑ older messages lost to context

Messages scroll out of context as your conversation grows.

A Fuller Context,
More Inaccurate the Response

As context fills with stale history, unrelated content, and repeated instructions, model accuracy on retrieval tasks drops sharply — even with the largest windows.

Source: OpenAI GPT-5.4 evals · MRCR v2 8-needle · March 2026

What's Inside a Skill?

The HOW

A Skill is just a markdown file with a specific structure. Here's the blueprint.

SKILL.md
Skill Name
code-review-assistant
name (required): • kebab-case only • No spaces or capitals • Should match folder name
Description
Use this skill when the user asks for a code review, PR feedback, or wants to improve code quality.
description (required): • MUST include BOTH: – What the skill does – When to use it (trigger conditions) • Under 1024 characters • No XML tags (< or >) • Include specific tasks users might say • Mention file types if relevant
Instructions
- Check for security vulnerabilities first - Evaluate naming conventions and readability - Suggest performance improvements - Always explain why something should change - Use the project's existing patterns as reference
The actual expertise. Write these as if you're onboarding a brilliant new teammate. Be concrete, not vague.

The Skills Folder Structure

Skills are stored in a specific directory that Claude automatically scans.

Personal / Universal Skills
macOS /Users/<your-username>/.claude/skills/
Linux /home/<your-username>/.claude/skills/
Windows C:\Users\<your-username>\.claude\skills\
Project Skills
Project skills (<repo>/.claude/skills/) live inside the repo. They follow the project — available to anyone who clones the repo. These are for project-specific conventions.
Claude.ai App
When using the Claude app, upload skill files under Customisation in your account settings.

Create Your First Skill in 3 Steps

From zero to a working skill in under 2 minutes.

1
Create the folder
$ mkdir -p ~/.claude/skills/user/my-skill

Name it something descriptive. One folder per skill.

2
Write your SKILL.md
SKILL.md
Skill Name: [Your Skill Name]DescriptionUse this skill when [describe the trigger conditions].Instructions- [Your specific instructions] - [Be concrete and actionable] - [Include examples of good output]

The description field is critical — it's how Claude decides when to use your skill.

3
Use it
Review this function for me
✨ Skill activated: my-skill
I'll review this using your code review skill. Checking security vulnerabilities first, then readability...

Just start chatting. Claude reads the SKILL.md automatically when your request matches the description.

Pro Tips

Be specific in your Description — vague triggers mean Claude won't know when to activate the skill.

Keep instructions concise. Skills that are too long can themselves consume context.

Test your skill by asking Claude something that should trigger it, then something that shouldn't.

Iterate. Your first version won't be perfect. Refine instructions based on Claude's actual responses.

One skill per folder. Don't cram multiple purposes into a single SKILL.md.

Keep descriptions specific and non-overlapping. It's the best way to keep things efficient.
